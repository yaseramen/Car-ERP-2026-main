# مرجع تشغيل ومستودع مشروع EFCT

> وثيقة للاحتفاظ بها: أين البُنى، أين الاستضافة، كيف تسحب الكود وتشغّل المشروع.  
> آخر تحديث يدوي للهيكل العام — راجع `package.json` و`README.md` لأرقام الإصدارات الدقيقة.

---

## 1. هوية المشروع

| البند | التفاصيل |
|--------|-----------|
| الاسم الظاهر للمنتج | **EFCT** |
| اسم الحزمة في `package.json` | `alameen-car-services` (قديم، لا يؤثر على التشغيل) |
| النوع | تطبيق ويب **SaaS** — إدارة مراكز خدمة سيارات |
| المستودع (Git) | عادةً على GitHub تحت حسابك؛ استنساخ: `git clone <رابط-المستودع>.git` |
| الفرع الافتراضي للتطوير | غالباً **`main`** (يتبع إعداد فريقك) |

---

## 2. البنية التقنية (أين بُني التطبيق)

| الطبقة | التقنية |
|--------|---------|
| الإطار | **Next.js** (App Router — مجلد `src/app/`) |
| اللغة | **TypeScript** |
| الواجهة | **Tailwind CSS** |
| قاعدة البيانات | **Turso** (SQLite عبر LibSQL) — مخطط في `database/schema.sql` وترحيلات في `database/migrations/` |
| المصادقة | **NextAuth (Auth.js v5)** — ملف `src/auth.ts` |
| الـ API | مسارات **Route Handlers** داخل `src/app/api/` (ليست خدمة خارجية منفصلة؛ جزء من نفس التطبيق) |

مسارات مهمة في الكود:

- `src/app/(auth)/` — تسجيل الدخول، التسجيل، استعادة كلمة مرور المالك
- `src/app/(dashboard)/admin/` — لوحة التحكم (فواتير، مخزن، ورشة، …)
- أنواع نشاط الشركة (`companies.business_type`): `both` | `sales_only` | `service_only` | `supplier` (مورّد = بدون ورشة/OBD، مثل محل القطع). أعلام السوق: `marketplace_enabled`, `ads_globally_disabled` — ترحيل `027_supplier_marketplace_flags.sql`.
- السوق: `028`–`030` (نوع محفظة `marketplace_ad`، جداول الباقات والإعلانات، شاشة `marketplace`). صفحة عامة `/market`، لوحة `/admin/marketplace`، Cron انتهاء الإعلانات `GET /api/cron/marketplace-expiry` مع `CRON_SECRET` (انظر `vercel.json`؛ على خطة Vercel Hobby الجدولة **مرة يومياً** كحد أقصى).
- **خارطة الطريق (ما بُني + ما التالي):** `docs/PRODUCT_ROADMAP.md`
- `src/lib/` — منطق مشترك (صلاحيات، محفظة، نسخ احتياطي، …)

## 3. الاستضافة والنشر (أين يعمل الموقع)

| البند | الملاحظة |
|--------|-----------|
| الاستضافة المقترحة في المشروع | **Vercel** (مناسب لـ Next.js؛ يذكر في `README.md`) |
| قاعدة البيانات | **Turso** — السحابة؛ ليست على نفس سيرفر Vercel |
| النطاق | يُضبط في Vercel + DNS؛ للنطاق المخصص يُذكر استخدام `NEXT_PUBLIC_APP_URL` في `README.md` |
| البناء على Vercel | يحتاج متغيرات البيئة (انظر القسم 5). أثناء `npm run build` يُشغَّل `db:migrate` تلقائياً على Turso (نفس متغيرات `TURSO_*` في Vercel)؛ إن فشل الترحيل يفشل البناء ويظهر السبب في سجلات النشر. |

**لا يوجد «استضافة API» منفصلة:** الـ API هي نفس تطبيق Next.js على Vercel (أو أي مضيف يشغّل `next start`).

## 4. سحب الكود والعمل محلياً

```bash
git clone <رابط-المستودع>.git
cd <مجلد-المشروع>
npm install
cp .env.example .env
# عدّل .env بالقيم الحقيقية (Turso + AUTH_SECRET + …)
npm run db:migrate
npm run dev
```

- التطوير المحلي: عادةً `http://localhost:3000`
- سكربتات إضافية: انظر `package.json` (`db:seed`, `build`, …)

---

## 5. متغيرات البيئة (ضرورية — لا تُرفع إلى Git)

| المتغير | الغرض |
|---------|--------|
| `TURSO_DATABASE_URL` | رابط قاعدة Turso |
| `TURSO_AUTH_TOKEN` | رمز مصادقة Turso |
| `AUTH_SECRET` أو `NEXTAUTH_SECRET` | سر توقيع الجلسات |
| `NEXTAUTH_URL` | رابط التطبيق (محلي أو إنتاج) |
| `NEXT_PUBLIC_APP_URL` | اختياري في الإنتاج لنطاق مخصص |

قالب كامل: `.env.example` في جذر المشروع.

**على Vercel:** نفس المتغيرات في **Settings → Environment Variables** (Production / Preview حسب الحاجة).

## 6. قاعدة البيانات والترحيل

- **المخطط المرجعي:** `database/schema.sql` (يُنفَّذ عبر سكربت الترحيل)
- **التغييرات التدريجية:** ملفات `database/migrations/*.sql` بالترتيب. منع تكرار كود/باركود الصنف لكل شركة: **في الـ API** (ليس فهرساً فريداً في DB — ترحيل `032` يحذف أي فهرس قديم من محاولة سابقة).
- الأمر: `npm run db:migrate` (يستخدم `scripts/migrate.ts` عبر `tsx`)

---

## 7. مستخدم Super Admin والنسخ الاحتياطي

- بذور أولية وكلمة مرور: انظر `README.md` (قسم الإعداد و `db:seed`)
- **نسيت كلمة مرور السوبر أدمن (طوارئ من السيرفر):**  
  `npx tsx scripts/reset-super-admin-password.ts <email> <new_password>`  
  (يتطلب `TURSO_*` في `.env`)

## 9. ماذا تحتفظ به شخصياً (خارج Git)

- نسخة آمنة من **`.env`** أو على الأقل: `TURSO_*`, `AUTH_SECRET`, روابط النطاق
- **رابط مستودع GitHub** واسم الفرع المعتاد
- **حساب Turso** ورابط لوحة التحكم
- **حساب Vercel** والمشروع المرتبط بالريبو
- بيانات **النطاق وDNS** إن وُجدت
